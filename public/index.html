WhatsApp Cloud API & n8n

This simulator displays exactly how the self-hosted n8n workflow processes messages sent to the Meta WhatsApp Cloud API.

Webhook Node (Trigger)

Receives the incoming messages from Meta and forwards them to n8n.

Switch Node (Logic)

Analyzes user input. Identifies if they selected a service ID (like srv_pancard) or typed standard terms, routing them to the correct reply node.

HTTP Request Node (Action)

Sends custom messages or interactive elements (like lists and buttons) back to the user via Meta's HTTP endpoint.

Setup Instructions (Hinglish / English)

1
Step 1: Meta Developer Account Setup

Go to Meta for Developers, log in with your client's Facebook account, and create a new App. Choose **Other** -> **Business** app type.

Hinglish: Meta for Developers par client ki Business ID se login karke ek App banayein aur usme **WhatsApp Business Platform** add karein.

2
Step 2: Get WhatsApp Credentials

In the WhatsApp Setup Panel, configure a phone number. Meta will provide a Temporary Access Token, a Phone Number ID, and a WhatsApp Business Account ID. Generate a Permanent Access Token in your System User settings to avoid authorization expiry.

Hinglish: WhatsApp settings me test number ya permanent client phone verify karein. Wahan se Phone Number ID copy kar lein aur System User section me ja kar ek **Permanent Access Token** generate karein jo kabhi expire nahi hoga.

3
Step 3: Setup n8n & Import Workflow

Deploy n8n (e.g. self-host on a cheap VPS or run it locally). Create a new workflow, and click Import from File. Upload the downloaded n8n_whatsapp_workflow.json.

Hinglish: n8n console open karein, Settings menu se 'Import from File' select karein, aur hamara download kiya hua JSON upload karein. Poora workflow diagram aapke saamne aa jayega.

4
Step 4: Update Workflow Credentials

Double-click the HTTP Request nodes in n8n and update:

  • The URL containing your Phone Number ID.
  • The Authorization Header containing your Permanent Access Token.
  • Update the website URLs in the JSON body payloads to point to your live site (e.g., replacement of https://www.yourcybercafewebsite.com with your domain).

Hinglish: n8n workflow me jo HTTP nodes hain, unme YOUR_PHONE_NUMBER_ID aur Authorization header me YOUR_PERMANENT_ACCESS_TOKEN daal kar update karein. Saath hi details me website url ko apne real web-app URL se change kar dein.

5
Step 5: Setup Webhooks in Meta Developer Dashboard

In n8n, activate the workflow. Copy the Production Webhook URL from the Webhook node (e.g., https://n8n.domain.com/webhook/whatsapp-webhook). In your Meta App Dashboard under WhatsApp Configuration, paste this URL and set your Verify Token (e.g., cybercafe_token). Finally, subscribe to messages in the Webhooks list.

Hinglish: n8n Webhook Node ka live URL copy karke Meta Developer Portal ke Webhooks Configuration me paste karein. Verify token jo set kiya hai wahi daalein. Configuration approve hone par **messages** event ko subscribe kar lein. Chatbot live ho jayega!
